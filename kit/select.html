Select

Basic

Description

Default select component with no attributes. Options provided via inline select element with data-fn="data-source".

Expectation

Component should:

  • Display a button showing default placeholder text "Select an option"
  • Show all options when clicked
  • Close dropdown and update button text when an option is selected
  • Support keyboard navigation (Arrow Up/Down, Enter, Home, End)
  • Have proper ARIA attributes (role="combobox", aria-haspopup, aria-expanded)
  • Auto-close dropdown after selection (single select mode)
Status Passed
Notes -

Attr: value="banana"

Description

Select component with pre-selected value via value attribute.

Expectation

Component should:

  • Display "Banana" as the selected option on load
  • Show checkmark next to "Banana" when dropdown is opened
  • Create hidden input with name and value for form submission
  • Value attribute should override any selected attributes in option elements
Status Passed
Notes -

Attr: placeholder="Choose a fruit"

Description

Select component with custom placeholder text.
Expectation

Component should:

  • Display "Choose a fruit" instead of default "Select an option"
  • Placeholder should be visible only when no option is selected
  • After selecting an option, show the selected option text
  • Placeholder should reappear if selection is cleared (in future implementations)
Status Passed
Notes -

Attr: disabled

Description

Select component in disabled state with pre-selected value.

Expectation

Component should:

  • Display the selected value but prevent all interactions
  • Button should have disabled attribute
  • Not open dropdown when clicked
  • Not respond to keyboard navigation
  • Not be focusable via tab navigation
  • Visual indication of disabled state (grayed out, cursor not-allowed)
Status Passed
Notes -

Attr: required

Description

Select component marked as required for form validation.
Expectation

Component should:

  • Integrate with HTML5 form validation
  • Prevent form submission if no option is selected
  • Show validation error message on submit without selection
  • Hidden input should have required attribute
  • Validation should clear once an option is selected
Status Passed
Notes Test within a form with submit button to verify validation behavior.

Attr: name="favorite-fruit"

Description

Select component with name attribute for form submission. Inspect DOM to verify hidden input creation.

Expectation

Component should:

  • Create a hidden input with name="favorite-fruit"
  • Hidden input value should update when selection changes
  • Form submission should include the selected value with correct name
  • Hidden input should be inspectable in DevTools
  • Work with standard form serialization (FormData API)
Status Passed
Notes Use browser DevTools to inspect hidden input element and verify name/value attributes.
